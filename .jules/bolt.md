## 2026-06-30 - Python sqlite3 executemany() Last Insert ID Anti-Pattern
**Learning:** When trying to optimize batch inserts in Python's `sqlite3` using `executemany()`, the driver sets `cur.lastrowid` to `None`. This prevents easily returning a list of inserted IDs in standard code flows. Additionally, generating identical timestamps quickly via loops can break regression tests requiring strictly unique per-row timestamps.
**Action:** To correctly return auto-incremented IDs after an `executemany()` operation without a `RETURNING` clause, execute `SELECT last_insert_rowid()` immediately after the insert, then calculate the batch of inserted IDs backwards using `cur.rowcount` (e.g., `first_id = last_id - rowcount + 1`). For unique timestamps, ensure an incremental offset like `time.time() + (i * 1e-6)` is applied per parameter tuple in the batch.
