## 2026-10-25 - [Python SQLite Batch Auto-increment Inserts]
**Learning:** `executemany` in Python `sqlite3` driver resets `cur.lastrowid` to `None`. To fetch the sequential auto-incremented primary keys generated for an entire batch without using a SQLite `RETURNING` clause (which may not always be supported or efficient depending on version/config), you can run a single `SELECT last_insert_rowid()` post-insert to obtain the maximum ID. By subtracting the size of the batch (using `len(params)`) from this maximum ID, the full sequentially assigned list of ids can be derived backward efficiently: `first_id = last_id - len(params) + 1`. This safely allows bulk batch insert speeds while still returning precise generated primary keys back to the application logic.
**Action:** Use this `last_insert_rowid()` backward-extrapolation pattern for any SQLite bulk insert operation (`executemany`) in the codebase that needs to retrieve inserted auto-increment IDs.
