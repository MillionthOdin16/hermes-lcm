## 2026-07-02 - SQLite executemany Performance and Timestamps
**Learning:** Python's `sqlite3` driver sets `cur.lastrowid` to `None` (or just the last inserted row) after `executemany()`. To retrieve auto-incremented IDs for batch inserts without a `RETURNING` clause, you must execute `SELECT last_insert_rowid()` immediately after the batch insert to get the last inserted ID, then calculate the preceding inserted IDs backward using `cur.rowcount`. Also, using `time.time()` once for the whole batch breaks the `test_append_batch_timestamps_are_unique_per_row` regression test.
**Action:** When optimizing batch inserts with `executemany` in this codebase, always apply a minor offset (`time.time() + (i * 1e-6)`) to timestamps, and calculate row IDs backwards using `rowcount` and `last_insert_rowid()`.
