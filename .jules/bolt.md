## 2026-05-18 - SQLite Batch Insert Optimization and ID Assignment
**Learning:** Python's SQLite `executemany` method natively suppresses the `lastrowid` return attribute, meaning it's unavailable immediately after a batch insert without specific handling. A batch insertion loop leveraging `execute` can cause performance degradation compared to `executemany`. However, the loss of `lastrowid` poses a challenge when trying to retrieve auto-generated IDs sequentially from the store. Further, for regression tests expecting distinct timestamps across rows (like `test_append_batch_timestamps_are_unique_per_row`), pre-calculating slightly incremented timestamps is required.
**Action:** When converting multiple `execute` inserts to a single `executemany` call for optimization, assign distinct timestamps inside a parameter loop prior to execution. Post-insertion, extract the final primary ID using a `SELECT last_insert_rowid()` query immediately after `executemany`, then calculate backwards using `cur.rowcount` to determine the full list of inserted sequential primary IDs.
