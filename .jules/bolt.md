## 2026-10-27 - SQLite executemany returning ID Optimization
**Learning:** Python's `sqlite3` driver resets `cur.lastrowid` to `None` when running `executemany()`. To capture auto-incremented primary keys for a batch without using `RETURNING` (which could cause version compatibility issues), we can fetch the max ID via `SELECT last_insert_rowid()` right after `executemany()` and calculate the IDs backward based on the batch size, since SQLite executes `executemany` sequentially in the same transaction.
**Action:** Use this `executemany` pattern paired with `SELECT last_insert_rowid()` and offset computation whenever inserting batches and needing their generated IDs. Also, when tests enforce strictly unique timestamps across rows in a batch, apply a small deterministic offset (e.g., `+ (i * 1e-6)`) to a single base `time.time()` reading.
