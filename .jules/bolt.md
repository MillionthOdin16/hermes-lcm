## 2026-04-18 - [SQLite Batch Insert Optimization]
**Learning:** Python's `sqlite3` driver sets `cur.lastrowid` to `None` after `executemany()`. To retrieve auto-incremented IDs for batch inserts without a `RETURNING` clause, execute `SELECT last_insert_rowid()` immediately after the batch insert to get the last inserted ID, then calculate the preceding inserted IDs backward using `cur.rowcount` or `len(batch)`. Also, regression constraints in this codebase may require unique timestamps per row, which can be accomplished by adding `i * 1e-6` to each timestamp.
**Action:** Always fetch `last_insert_rowid()` following `executemany` to infer generated IDs. Ensure generated timestamps in loop preserve strict monotonic ordering via artificial offset when inserting message batches in this codebase.
