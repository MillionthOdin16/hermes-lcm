## 2026-05-18 - [SQLite Optimization Pattern: Batch Inserts with executemany]
**Learning:** Python's `sqlite3` driver sets `cur.lastrowid` to `None` after `executemany()`. To retrieve auto-incremented IDs for batch inserts without parsing a `RETURNING` clause result set, it's possible to execute `SELECT last_insert_rowid()` immediately after the batch insert to get the last inserted ID, and calculate the preceding inserted IDs backward using `cur.rowcount` (e.g. `last_id - cur.rowcount + 1` up to `last_id`).
**Action:** Use this pattern to dramatically speed up batch inserts (`_append_protected_batch`) that require returning generated keys while ensuring backwards compatibility with older sqlite versions that lack `RETURNING`. Also apply minor offset offsets (e.g. `1e-6`) to timestamps during batch prep to respect unique-timestamp constraints.
