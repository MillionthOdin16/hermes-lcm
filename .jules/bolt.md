## 2026-06-29 - SQLite executemany lastrowid missing
**Learning:** Python's standard `sqlite3` sets `cur.lastrowid` to `None` after `executemany()`. The codebase relies on returning inserted auto-incremented IDs to callers for store inserts. Since standard SQLite driver doesn't natively return list of inserted IDs on executemany without RETURNING, an optimization that just relies on `cur.lastrowid` will break or return None.
**Action:** When using `executemany` for auto-incremented batch inserts in this codebase, immediately run `SELECT last_insert_rowid()` afterwards. Because SQLite issues sequential row IDs for batch inserts within the same transaction without holes, you can calculate the entire batch's inserted IDs backward by doing `first_id = last_id - len(params) + 1` and returning `list(range(first_id, last_id + 1))`. This allows massive optimization of `store.py` inserts while preserving the function signature of returning an ID list.
