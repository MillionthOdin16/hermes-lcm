## 2026-06-21 - Python `_row_to_dict` Performance Anti-Pattern
**Learning:** In a hot path retrieving data from an SQLite database, converting a row to a dictionary using `cols = [...]` list allocation and `dict(zip(cols, row))` is significantly slower than direct manual dictionary creation from tuple indices. Additionally, performing function calls like `_normalize_source_value` inside the row parsing hot loop introduces massive overhead compared to inlining the logic.
**Action:** When working with SQLite fetch operations or tight data transformation loops, avoid intermediate allocations (like lists for keys) and dynamic constructs like `zip`. Hardcode dictionary construction directly using positional arguments and inline small helper functions to skip function call overhead.
